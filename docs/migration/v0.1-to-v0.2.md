### Resolution 2: Commit Boundary Enforcement

**v0.1 Problem:**
- Constitution required immutability of `author` and `committed_at` only
- API delegated enforcement to Policy (optional)
- Core immutability was not guaranteed

**v0.2 Solution:**

**Constitution 4.2:**
```
After commit, the Core MUST NOT allow modification of:
- metadata.author
- metadata.committed_at
- id

Implementations MUST NOT delegate solely to Policy layers.
```

**API v0.2:**

### Resolution 3: State Model Unification

**v0.1 Problem:**
- Formalization: `S = {Draft, Valid, Invalid, Committed}`
- Schema: `status: "active | superseded | deprecated"`
- Two different state systems with no mapping

**v0.2 Solution:**

**Constitution 7.1 & 7.2:**
```
Node Status: active | superseded | deprecated
Edge Status: active | superseded
Graph State: Draft | Valid | Invalid | Committed
```

**Clarification:**
- **Node/Edge status** = lifecycle state (part of data)
- **Graph state** = validation state (derived from lint)
- These are orthogonal dimensions, now explicitly defined

**Formalization v0.2:**
```
S_N = {active, superseded, deprecated}  (node status)
S_E = {active, superseded}              (edge status)
S_G = {Draft, Valid, Invalid, Committed} (graph state)
```

**Verification:** âœ… Clear separation of concerns

## Automated Migration Tool (Specification)

```typescript
interface MigrationResult {
  success: boolean;
  changes: {
    edges_with_ids_added: number;
    edges_with_status_added: number;
    nodes_failed_author_check: string[];
    status_values_normalized: number;
  };
  errors: Array<{
    node_id: string;
    error: string;
  }>;
}

async function migrateToV02(
  decisions: Decision[]
): Promise<MigrationResult> {
  const result: MigrationResult = {
    success: true,
    changes: {
      edges_with_ids_added: 0,
      edges_with_status_added: 0,
      nodes_failed_author_check: [],
      status_values_normalized: 0
    },
    errors: []
  };

  for (const decision of decisions) {
    // 1. Validate author
    if (!decision.metadata?.author) {
      result.errors.push({
        node_id: decision.id,
        error: "Missing required field: metadata.author"
      });
      result.success = false;
      continue;
    }

    // 2. Normalize status
    if (!["active", "superseded", "deprecated"].includes(decision.status)) {
      decision.status = "active";
      result.changes.status_values_normalized++;
    }

    // 3. Add edge IDs
    decision.edges = decision.edges.map((edge, index) => {
      if (!edge.id) {
        edge.id = `edge-${decision.id}-${index}`;
        result.changes.edges_with_ids_added++;
      }
      if (!edge.status) {
        edge.status = "active";
        result.changes.edges_with_status_added++;
      }
      return edge;
    });
  }

  return result;
}
```

## Breaking Changes Communication

### For Users

**You MUST update if:**
- You have decisions without `metadata.author`
- You use `remove_edge` operations
- Your edges lack `id` or `status` fields
- You rely on status values outside `{active, superseded, deprecated}`

**Migration is required.**
v0.1 graphs are **not** forward-compatible without migration.

---

### For Implementers

**You MUST update if:**
- Your kernel allows edge deletion
- You delegate immutability to policy only
- You don't enforce `author` requirement
- Your validation doesn't check edge `id` and `status`

**Constitutional compliance is mandatory.**
Non-compliant implementations cannot claim v0.2 conformance.
